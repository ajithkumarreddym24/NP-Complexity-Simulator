Unit 6 — Undecidability

The Halting Problem

Alan Turing's 1936 proof that no algorithm can decide whether an arbitrary program halts on a given input. Explore the diagonal argument and understand undecidability through interactive simulation.

Cantor's Diagonal Argument — Visualized

Imagine a hypothetical HALT decider H(P,I) — Turing's proof shows this leads to a contradiction

The Contradiction Proof

1
Assume HALT Decider Exists
function H(program P, input I): if P halts on I: return TRUE else: return FALSE
2
Construct Contradictory Program D
function D(program P): if H(P, P) == TRUE: loop_forever() // ← Contradiction! else: return "halts" // ← Contradiction!
3
Run D on itself: D(D)
D(D): if H(D, D) == TRUE // D halts on D? → D loops forever // CONTRADICTION ✗ if H(D, D) == FALSE // D doesn't halt? → D halts // CONTRADICTION ✗
∴
H cannot exist — Halting Problem is Undecidable

No Turing Machine can solve the Halting Problem for all inputs. This is stronger than NP-Hard — it is uncomputable.
